关于 border-radius 自己要注意的几点

标签（空格分隔）： css

关于 border-radius
Safari 浏览器、Firefox 浏览器、Opera 浏览器及 Chrome 浏览器都支持这种绘制圆角边框的样式。

但是针对不同浏览器的写法要注意:
要注意添加上浏览器前缀;

- Firefox 为'-moz-border-radius'的形式
- Opera 为 'border-radius'的形式
- Chrome 为 'border-radius' 或 '-webkit-border-radius' 的形式。

在 border-radius 属性中，可以指定两个半径。
在 Chrome 浏览器与 Safari 浏览器中，会绘制出一个椭圆形边框，一个半径为椭圆的水平方向半径，第二个半径为椭圆的垂直方向半径。
但是在 Firefox 浏览器与 Opera 浏览器中，将第一个半径作为边框左上角与右下角的圆半径来绘制，将第二个半径作为边框右上角与左下角的圆半径来绘制。

boder 设置为 none ，并不影响 border-radius 的显示

利用图床加速

标签（空格分隔）： 优化

最近做的招新网页，整个网页都是由图片和动画构成的，刚做好，把整个项目放到服务器的时候，发现，整个网页加载得很慢。用户体验很不好，后来想到了七牛，之前有看到师兄们涉及到图片这一块的时候会去用七牛，所以就去了解了一下，了解到了一种关于图床的东西。
图床专门用来存放图片，同时允许你把图片对外连接的网上空间。之前写的博文也有用七牛存储去获取图片的对外链接，但一直不知道到图床在网页的加载这一方面的优势，尤其是拥有大量图片的网页。
这里说的是七牛，七牛主要是利用 CDN 内容分发网络来处理内容的传输，我们把图片存储到第三方CDN，除了加快图片访问速度，还可以降低我们的服务器的存储压力，而且后续网站迁移扩容也不会受影响，另外，不同的宽带提供商的用户访问图床都会比访问服务器要快。
在用了七牛后，确实整个网页顺畅了很多，其他优化的东西还要继续学习。
